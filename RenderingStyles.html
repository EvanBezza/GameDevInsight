Rendering Techniques in Games

There are loads of ways to improve a games performance by using literal asset managment scripts such as object pooling and such, however I will only speak of two methods in here, Level of Detail (LOD) and Occlusion Culling.

Level of Detail Meshes

LOD is used to set various levels of complexity and detail for objects and assets in games. Developers use this method to greatly reduse hardware strain a game has on a computer by reducing the level of detail a object or asset has the further a players camera is from the object.

For example, if the player is 15 meters from an object, the object will have a LOD of 1 which has a low polly version of that object, then if the player is say 5 meters from the object it'll have an LOD of 0 which is the original high rendered model of the object.

This is a great way to increase performance of a game and if you'd like a more in depth insight on this procees, feel free to follow this link to Unity's documentation on LOD Meshes.

Occlusion Culling

Occlusion Culling is a key component on greatly increasing the performance of any game you make,

This process improves performance by only rendering assets and objects that are within the view of the players camera. So if an object or level is behind the player it is essentially temporarily deleted and then brought back as the camera approaches it.

If you'd like to read more about this Occlusion Culling techniques, feel free to read about it on Unity's documentation about Occlusion Culling.
